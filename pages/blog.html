A Look into DNS

  Let's start with what DNS is. DNS is the system that translates human-readable domain names to an Internet Protocol Address. Depending on how the configuration, a domain may point to multiple IP addresses. Let's break down the problem?
How does a computer resolve these DNS requests? The OSI Transport Layer, formally known as the fourth layer of the model, sends out a User Datagram query to a DNS Server. A recursive query will check with a DNS Server and continue querying other DNS servers to find the IP address associated with the domain. It could check the local DNS cache, the Root DNS, or the Authoritative DNS server. Once it receives the response to the correct domain, it will send out the page request and the process is done.

The DNS query has two parts. The header and the question. The header consists of twelve bytes. Let's break down each of the bits.

DNS Header
ID - 16 bits (2 bytes) - This part of the header signifies a assigned number by the system that is creating the query, and is mirrored in the DNS response.
QR - 1 bit - This represents whether this message is a query or ar esponse.
Opcode - 4 bits - Which type of query this is (0 is standard, 1 is inverse, 2 is server status).
AA TC RD RA - 1 bit each - Authoriative Answer, bit will default at 0, if its an authoritative answer, the bit will be set. TC signifies that the message has been truncated because the length is greater than the required length in the transmission channel. RD and RA are the bits that exhibit whether or not recursion is used or can be used.
QDCOUNT, ANCOUNT, NSCOUNT, ARCOUNT - 16 bits each -

Huffman Coding

      Huffman Code is a data compression algorithm that enables lossless compression. Lossless compression allows for the original data to be reconstructed without loss of any information. The algorithm entails finding the frequency of each character. We will then create a min-heap in which the parent node, the most frequent character, will be closer to the top of the tree. We will create leaf nodes for the least frequent characters. Then we must create an internal node, which is the sum of the frequencies of two of the minimum-frequency nodes. After this, we will build up the tree by adding nodes that show up more often as opposed to the less frequent characters, continuously creating internal nodes, until we ge get to the root of the heap, which is the total character count. The left child of the root will be the 0 bit, whereas the right will be given the 1 bit. To store the text, we will utilize our min heap, and follow the path to the character wherever it may be in the tree.


Example:
        

              18

         /            \

10(A)         8(Internal Node)

                      /                   \
                  4(B)                4 (C)


In this example, 10 will be set to bit 0.
The internal node will be set to 1
The 4(B) value will be set to 0.
The 4(C) value will be set to 1.

Now lastly, let's say we have the letters "ABAAC"?
We know that:
A = 0
B = 10
C = 11
Therefore, to translate "ABAAC", we can deduce that the Huffman encoding will be:
A = 0 || B = 10 || A = 0 || A = 0 || C == 11
The final product then becomes:
