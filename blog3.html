Huffman Coding

      Huffman Coding is a data compression algorithm that enables lossless compression, allowing the original data to be reconstructed without any loss of information. The algorithm starts by determining the frequency of each character in the input data. These frequencies are used to create a min-heap, where each node represents a character and its frequency, with nodes of lower frequencies having higher priority. The two nodes with the smallest frequencies are extracted from the min-heap and combined into a new internal node with a frequency equal to the sum of the two nodes. This new node is then reinserted into the heap. This process is repeated until only one node remains, forming the root of the Huffman tree. The Huffman tree is then traversed, assigning a '0' to left edges and a '1' to right edges, to generate Huffman codes for each character. These codes are then used to encode the input data, replacing each character with its corresponding Huffman code, thus achieving compression.


Example:
        

              18

         /           \

10(A)         8(Internal Node)

                      /       \
                  4(B)            4 (C)


In this example, 10 will be set to bit 0.
The internal node will be set to 1
The 4(B) value will be set to 0.
The 4(C) value will be set to 1.

Adjusted Example:

                 18

           /            \

(0)10(A)        (1) 8(Internal Node)

                      /         \
              ( 0) 4(B)          (1) 4(C)



Now lastly, let's say we have the letters "ABAAC"?
We know that:
A = 0
B = 10
C = 11
Therefore, to translate "ABAAC", we can deduce that the Huffman encoding will be:
A = 0 || B = 10 || A = 0 || A = 0 || C == 11
The final product then becomes:
0100011
